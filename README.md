# 總務處行政管理系統：系統架構設定提示詞

## 一、 專案基礎設定 (Project Configuration)
請於 GitHub Repository 的 Projects 功能中建立新專案，並設定以下欄位 (Fields)：
1. **Status (狀態)**：選項包含 `To Do` (待辦), `In Progress` (進行中), `Done` (已完成)。
2. **Assignee (人員)**：用於指派任務負責人。
3. **Due Date (日期)**：用於定義任務截止期限。
4. **Milestone (里程碑)**：用於歸類年度專案（例如：暑期修繕、感恩餐會）。
5. **Label (標籤 - 核心數據)**：必須建立以下標籤以進行績效統計：
   * `權重:1` (輕便任務)
   * `權重:3` (中型任務)
   * `權重:5` (大型任務)

## 二、 視圖配置 (View Setup)
在 Project 中建立以下四個視圖，並分別設定檢視模式：
1. **看板視圖 (Board)**：設定 Group by Status。用於每日任務的快速拖曳更新。
2. **時間軸視圖 (Timeline)**：設定 Group by Assignee。用於人員負荷預覽與排程規劃。
3. **計畫視圖 (Milestone)**：設定 Filter 為 Milestone。用於追蹤年度重點計畫的進度條。
4. **統計圖表 (Insights)**：設定 Chart 為 Bar Chart：
   * **X 軸 (Group by)**：Assignee
   * **Y 軸 (Value)**：Sum of Label (權重)
   * 此圖表將自動計算每位人員的「累積工作貢獻值」。

## 三、 作業 SOP (Operation Workflow)
* **計畫階段**：主任利用「時間軸視圖」配置人力資源，設定截止日期與里程碑。
* **執行階段**：同仁透過「看板視圖」領取任務，移動至 `In Progress`，完工後移動至 `Done`。
* **監控階段**：主任切換至「計畫視圖」，檢視年度專案進度條。
* **考核階段**：每月底切換至「統計圖表」，匯出或截圖各人員的累積權重貢獻，作為績效分析依據。

---

💡 **提示**：
此架構完全免除程式維護，透過 GitHub Projects 的原生配置 即可達成「規劃、排程、執行、考核」的閉環管理。若要調整工作量權重，僅需修改 Label 標籤設定，所有圖表數據將自動同步重新計算。
